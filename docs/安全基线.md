# KMS 安全基线

更新时间：2026-04-29

本文档仅维护“当前安全基线（必须保持）”。  
后续新增的安全决策，统一按模块追加到本文件。

## 当前安全基线（必须保持）

### 模块：会话与令牌

1. Access Token 与前端会话 Cookie TTL 必须统一为 12 小时。
2. Token 必须包含 `sub`、`exp`、`ver`、`jti`。
3. Token 撤销必须双机制并存：`token_version`（批量失效）+ `jti`（单 token 失效）。
4. 用户密级、启停状态、密码变更后，必须触发 `token_version` 递增。
5. 必须提供后端注销接口并在注销时吊销当前 Token 的 `jti`。

### 模块：登录防护

1. 登录错误上限必须为 5 次。
2. 第 5 次错误触发锁定 5 分钟。
3. 锁定期间不处理密码校验，不累计额外惩罚，只返回锁定状态与解锁时间。
4. 锁定结束后再次输错，锁定时长在上次基础上额外 +5 分钟（第 6 次错锁 10 分钟，第 7 次错锁 15 分钟，以此类推）。
5. 成功登录后必须重置失败计数并清除锁定状态。
6. 登录失败响应需包含可用于前端提示的结构化信息（错误码、剩余次数、锁定截止时间）。
7. `locked_until` 必须返回带时区的 ISO 8601 绝对时间，前端统一按北京时间展示。

### 模块：401 会话清理

1. 后端返回 `401` 时，前端必须统一进入清会话流程。
2. 清会话流程必须是“先调用后端注销，再清除本地 Cookie”。
3. 前端受保护路由与附件代理路由出现 `401` 时，必须走 `/logout`，不得仅跳转 `/login`。

### 模块：CORS 与环境安全

1. CORS 仅允许白名单，不允许通配 `*`。
2. CORS 白名单必须支持后台在线维护。
3. CORS 白名单管理接口权限仅限 `admin`。
4. 生产环境必须拒绝默认安全配置启动：
5. `SECRET_KEY=change-me` 时禁止启动。
6. CORS 保持 localhost 默认白名单时禁止启动。
7. CORS 白名单为空时禁止启动。

### 模块：鉴权与审计

1. 统一通过后端鉴权依赖解析 Bearer Token 并执行角色校验。
2. 鉴权失败必须返回明确 `401/403`，不得降级放行。
3. 必须保留认证/鉴权审计日志，至少覆盖登录成功/失败、注销成功/失败、Bearer 鉴权失败、角色鉴权拒绝。
4. 审计日志必须可由后台查询，且查询权限仅限 `admin`。

### 模块：内容编辑权限

1. 管理员必须可编辑所有笔记。
2. 笔记创建者必须可编辑自己的笔记，并可选择编辑策略。
3. 笔记编辑策略固定为两类：仅创建者和管理员可编辑；或达到该笔记密级及以上的用户可编辑。
4. 非创建者不得修改笔记编辑策略，即使该用户因公开编辑策略获得内容编辑权限。

### 模块：账户与密码策略

1. 当前角色模型固定为 `admin` 与 `employee` 两类。
2. 员工默认密码可为种子策略指定值，但“用户主动修改密码”必须满足复杂度规则：6-64 位，且包含字母与数字。
3. 首次登录不强制改密，但必须通过消息红点提醒未改密状态（`need_password_change=true`）。
4. 员工在个人中心可修改密码。
5. 员工可在登录页提交密码重置申请；后台必须记录待处理申请，供管理员查看。
6. 管理员后台仅可将员工密码重置为系统默认值 `123456`，不得自定义员工明文密码。
7. 管理员重置员工密码时，必须设置 `need_password_change=true`、清除登录锁定状态、递增 `token_version`，并写入认证审计日志。

### 模块：组织与账号生命周期

1. 必须保留部门模型（`departments`）并支持与用户关联。
2. 员工档案必须包含：姓名、部门、职位、性别、简介、电话、邮箱、创建时间、停用时间。
3. 管理员创建员工时电话和邮箱可留空；员工可在个人中心补充，管理员可修改员工资料。
4. 管理员查询员工必须支持按部门、关键词（姓名/账号等）、账号状态筛选。
5. 账号停用时必须记录停用时间；恢复启用时应清除停用时间并触发会话失效（`token_version` 递增）。

### 模块：开发态约束

1. 种子账号密码当前保持不变（`admin/123456`、`user/123456`），仅用于开发联调。
2. 任何涉及安全策略的代码改动，必须同步更新本文件对应模块条目。
