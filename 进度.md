# KMS MVP 进度看板

## 当前状态

- 项目目标：可上线 MVP
- 当前轮次：Iteration 002
- 当前重点：先完成主链路功能，再逐步补齐页面高保真还原
- 当前运行方式：

  - 后端启动时自动执行 Alembic 迁移
  - 前端使用 Next.js standalone 镜像运行

## 已确定方案

- 前端：Next.js
- 后端：FastAPI
- 数据库：MySQL
- 对象存储：MinIO
- 检索：Elasticsearch
- 权限模型：`admin + employee` 双角色，管理员分配员工密级（L1-L3），管理员为 L4
- 笔记存储：`TipTap JSON + content_text`
- 审核方向：规则 + 敏感词

## 当前判断（2026-04-06）

### 这个项目要做什么
- 这是一个企业知识管理系统（KMS）MVP。
- 核心目标是把企业知识按“知识仓库 -> 目录 -> 笔记 -> 附件”沉淀下来，并提供权限控制、全文检索、问答和后台管理。
- 目标交付内容包括：
  - 用户登录与权限控制
  - 知识仓库浏览、笔记阅读与编辑
  - PDF / DOCX 附件入库
  - Elasticsearch 全文检索
  - 基于权限过滤的 RAG 问答
  - 管理后台与基础部署能力

### 当前代码实际状态
- 项目不是空壳，主链路已经具备可演示的 MVP 骨架。
- 后端已实现 JWT 登录、当前用户、仓库/笔记读取、笔记编辑、附件上传、搜索、问答、后台 CRUD 等接口。
- 前端已实现仓库、仓库详情、笔记详情、笔记编辑、搜索、问答、后台等页面，并接上真实后端数据。
- 当前仍处于“开发态可联调”阶段，不等同于“正式用户态可上线”阶段。

### 本轮更新（2026-04-07，认证与鉴权加固）
- CORS 已改为运行时白名单机制，默认从环境变量加载，并可由后台在线更新后即时生效。
- 后台新增安全配置与审计读取能力：

- 生产环境启动保护已落地：
  - `SECRET_KEY=change-me` 时拒绝启动；
  - CORS 仍是 localhost 默认白名单时拒绝启动；
  - 生产环境空白名单拒绝启动。
- Token/Cookie TTL 已统一为 12 小时：
  - 后端 `ACCESS_TOKEN_EXPIRE_MINUTES=720`
  - 前端会话 Cookie `maxAge=12h`
- 登录失败次数限制已落地：
  - 连续密码错误 5 次后锁定账号（默认 5 分钟）

- 已实现 `jti + token_version` 双撤销机制：

  - 鉴权时校验 `token_version`
  - 新增 `revoked_tokens` 表，支持按 `jti` 精确吊销
  - 新增 `POST /api/v1/auth/logout`，退出时吊销当前 token 的 `jti`
- 前端 401 清理会话链路已闭环：
  - 页面鉴权与 API 请求遇 401 统一跳 `/logout`
  - `/logout` 先调后端注销，再清理本地 cookie
- 已新增认证审计模型与日志记录，覆盖登录/退出/鉴权失败/鉴权拒绝等关键事件。
- 修复 Alembic 在 MySQL 8.4 的兼容问题（`TEXT` 默认值非法），并将 `0002` 迁移改为幂等，避免“半迁移状态”导致重复字段报错。
- 新增专项文档 `docs/安全基线.md`，用于按模块沉淀“当前必须保持”的安全决策并约束后续改动流程。

### 编码与排查说明
- 当前开发环境使用 Windows PowerShell 5.1，默认读取 UTF-8 中文文件时可能出现乱码显示。
- 这类乱码优先视为终端读取编码问题，而不是源码一定损坏。
- 在终端查看中文代码或文档时，应显式使用 `Get-Content -Encoding UTF8`，避免误判。
