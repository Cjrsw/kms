# KMS MVP 进度看板

最后更新：2026-04-21

## 当前状态

项目后端主链路代码已基本成型，具备真实登录、权限控制、知识库管理、附件访问、全文检索、Hybrid Recall 问答、QA 流式输出以及后台管理能力。当前 `frontend` 已切换为 `web/PythonProject` 静态前端草稿，旧 Next.js 前端已备份到 `_archived/frontend-next-*`，前端真实业务接入需要重新逐步迁移。

当前阶段不是“从零开发”，而是“在已成型 MVP 上补齐最后几个关键短板并做联调收口”。

## 当前真实能力

### 已完成

- 认证与鉴权
  - 登录、登出、当前用户
  - 401 清会话闭环
  - 登录失败锁定
  - `jti + token_version` 失效机制
  - 认证与鉴权审计
  - 静态前端登录已接入 `/api/v1/auth/session-login`，通过 httpOnly `kms_access_token` Cookie 保持会话，并由 nginx 代理注入 Bearer 调用后端接口

- 用户与组织
  - `admin` / `employee` 双角色模型
  - 部门管理
  - 员工创建、查询、编辑、停用
  - 当前运行中的 `frontend` 是 `web/PythonProject` 静态草稿；旧 Next.js 用户侧、登录页、管理员后台已整体备份，后续需按草稿重新接入真实接口
  - 个人中心已拆分为概览页、资料编辑页、密码修改页、收藏列表页
  - 用户头像上传、替换、删除与展示

- 知识内容
  - 仓库、目录、笔记、附件读写
  - 笔记后端已支持 `content_markdown` 主正文；`content_text` 由 Markdown / 旧纯文本派生并继续服务搜索与 QA
  - 仓库封面图后台上传、替换、删除与前台卡片展示
  - 用户侧新增目录/笔记，删除入口收口为“侧边栏删除笔记”
  - 后台仓库/目录/笔记 CRUD
  - 普通员工仅可删除自己创建的笔记；管理员可删除任意笔记

  - 后台目录删除已改为定向清理，不再整库重建
  - 笔记密级与目录/仓库下限校验

- 附件能力
  - 上传、替换、删除、下载
  - PDF 在线预览

  - 新上传 PDF / DOCX 文本抽取
  - 附件上传 / 替换已走 Celery 异步入库；后台完成解析、提取文本、ES/Qdrant 重建，并回写索引状态

  - Elasticsearch chunk 级索引
  - 关键词检索
  - Qdrant 向量检索

  - 权限过滤

  - `citation_status`

  - GPT 风格 QA 页面（左侧历史 + 右侧消息流）
  - 后台 `Sys Prompt` 配置
  - 后台 QA 审计
  - 结构感知切分 + 句子级拼块 + 小重叠窗口

    - 认证审计
    - Sys Prompt

  - 后台总览已收口为轻量概览页，仅保留统计卡片
  - 分析型大盘暂未推进，原因是当前后端尚未稳定记录首字延迟、Token 消耗等字段
  - 后台仓库管理页已切到轻量仓库列表接口，不再为仓库弹窗和列表页加载整份 `/admin/content`

### 已禁用或已收口

- 管理员自由注册/切换模型

- 运行时从数据库动态选 chat / embedding 模型

当前 AI 运行策略是固定模型配置：chat 和 embedding 统一从 `.env` 读取。

## 当前阻塞

### 阻塞 1：QA 结果稳定性仍受上游模型影响

真实状态：

- 本地代码链路已打通
- 智谱上游仍可能出现限流或超时

影响：

- QA 成功率和时延不完全由本地代码决定
- 本轮实测仍出现 `model_timeout`
- 单次时延波动仍较大，约 11s 到 60s+

### 阻塞 2：浏览器全量回归未完成

尚未做完整人工回归的模块：

- 登录锁定与错误提示
- 个人中心改密闭环
- 搜索筛选组合
- 后台用户/部门/Sys Prompt/QA 审计

说明：

- QA 页面本身已完成浏览器抽测
- 其余模块仍缺完整人工回归
