# KMS MVP 进度看板

## 当前状态

- 项目目标：可上线 MVP
- 当前轮次：Iteration 002
- 当前重点：先完成主链路功能，再逐步补齐页面高保真还原
- 当前运行方式：
  - 后端使用现有容器镜像 + 挂载源码 + `uvicorn`
  - 前端使用现有容器镜像 + 挂载源码 + standalone 启动
  - 这样做的原因是外部镜像代理偶发 `500`，暂时不适合依赖标准 `docker compose build frontend` 重建链路

## 已确定方案

- 前端：Next.js
- 后端：FastAPI
- 数据库：MySQL
- 对象存储：MinIO
- 检索：Elasticsearch
- 权限模型：管理员分配角色与密级，低级员工不能搜索或查看高级信息
- 笔记存储：`TipTap JSON + content_text`
- 审核方向：规则 + 敏感词

## 当前约束与说明

- 当前仓库是从空目录逐步搭建出来的，不是基于现成工程增量改造
- 前端为了联调方便，当前仍会自动使用开发态种子账号获取 token
- 数据库初始化目前使用 `create_all + seed`
- 终端里查看中文返回时可能出现编码显示异常，但服务本身是正常的
- 前端当前访问后端使用 `host.docker.internal:8000`，用于绕过容器内 `localhost` 指向错误的问题
