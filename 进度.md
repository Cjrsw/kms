# KMS MVP 进度看板

## 当前状态

- 项目目标：可上线 MVP
- 当前轮次：Iteration 002
- 当前重点：先完成主链路功能，再逐步补齐页面高保真还原
- 当前运行方式：
  - 后端使用现有容器镜像 + 挂载源码 + `uvicorn`
  - 前端使用现有容器镜像 + 挂载源码 + standalone 启动
  - 这样做的原因是外部镜像代理偶发 `500`，暂时不适合依赖标准 `docker compose build frontend` 重建链路

## 已确定方案

- 前端：Next.js
- 后端：FastAPI
- 数据库：MySQL
- 对象存储：MinIO
- 检索：Elasticsearch
- 权限模型：管理员分配角色与密级，低级员工不能搜索或查看高级信息
- 笔记存储：`TipTap JSON + content_text`
- 审核方向：规则 + 敏感词

## 当前判断（2026-04-05）

### 这个项目要做什么
- 这是一个企业知识管理系统（KMS）MVP。
- 核心目标是把企业知识按“知识仓库 -> 目录 -> 笔记 -> 附件”沉淀下来，并提供权限控制、全文检索、问答和后台管理。
- 目标交付内容包括：
  - 用户登录与权限控制
  - 知识仓库浏览、笔记阅读与编辑
  - PDF / DOCX 附件入库
  - Elasticsearch 全文检索
  - 基于权限过滤的 RAG 问答
  - 管理后台与基础部署能力

### 当前代码实际状态
- 项目不是空壳，主链路已经具备可演示的 MVP 骨架。
- 后端已实现 JWT 登录、当前用户、仓库/笔记读取、笔记编辑、附件上传、搜索、问答、后台 CRUD 等接口。
- 前端已实现仓库、仓库详情、笔记详情、笔记编辑、搜索、问答、后台等页面，并接上真实后端数据。
- 当前仍处于“开发态可联调”阶段，不等同于“正式用户态可上线”阶段。

### 本轮更新（2026-04-05）
- 已将 PowerShell 读取 UTF-8 中文文件的注意事项补充进 `任务.md`，避免后续因终端编码误判源码问题。
- 已开始将前端从“开发态种子账号自动换 token”切换到“真实登录 + 会话 cookie”模式。
- 已新增前端登录 action、退出登录路由、服务端会话辅助模块。
- 已将首页、仓库页、搜索页、问答页、后台页、笔记详情页、编辑页切到真实当前用户模型。
- 已把附件下载链路改为前端代理路由，避免真实登录后浏览器直接访问受保护后端接口失效。
- 本轮尚未完成前端构建与浏览器实测，因为当前机器缺少可直接执行的前端依赖链，Docker daemon 也未处于可用状态。

### 当前明确存在的问题
- 前端仍通过开发种子账号自动换取 token，尚未完成真实用户登录闭环。
- 附件下载目前直接访问受保护后端接口，切换到真实用户态后会有鉴权链路问题。
- 后台目前只有仓库、目录、笔记 CRUD，尚未覆盖用户、角色、密级管理。
- 搜索仍是整篇笔记级索引，尚未做到 chunk 级检索与更细粒度高亮。
- 问答仍是“检索后整理回答”的 MVP，不是真正的大模型生成。
- 数据库迁移仍是 `create_all + seed`，尚未切到 Alembic。
- Celery 与异步入库链路目前只有骨架，占位多于真实执行。

### 编码与排查说明
- 当前开发环境使用 Windows PowerShell 5.1，默认读取 UTF-8 中文文件时可能出现乱码显示。
- 这类乱码优先视为终端读取编码问题，而不是源码一定损坏。
- 在终端查看中文代码或文档时，应显式使用 `Get-Content -Encoding UTF8`，避免误判。
