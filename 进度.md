# KMS MVP 进度看板

最后更新：2026-04-18

## 当前状态

项目已经不是原型壳子，主链路代码已基本成型。当前系统具备真实登录、权限控制、知识库管理、附件访问、全文检索、Hybrid Recall 问答、QA 流式输出以及后台管理能力。

当前阶段不是“从零开发”，而是“在已成型 MVP 上补齐最后几个关键短板并做联调收口”。

## 当前真实能力

### 已完成

- 认证与鉴权
  - 登录、登出、当前用户
  - 401 清会话闭环
  - 登录失败锁定
  - `jti + token_version` 失效机制
  - 认证与鉴权审计

- 用户与组织
  - `admin` / `employee` 双角色模型
  - 部门管理
  - 员工创建、查询、编辑、停用
  - 个人中心资料维护与改密

- 知识内容
  - 仓库、目录、笔记、附件读写
  - 用户侧新增/删除笔记与目录
  - 后台仓库/目录/笔记 CRUD
  - 笔记密级与目录/仓库下限校验

- 附件能力
  - 上传、替换、删除、下载
  - PDF 在线预览
  - DOCX 提示后下载
  - 新上传 PDF / DOCX 文本抽取

- 搜索能力
  - Elasticsearch chunk 级索引
  - 关键词检索
  - Qdrant 向量检索
  - Hybrid Recall（ES + Qdrant）
  - 知识库 / 文件类型 / 时间 / 发布人员筛选
  - 搜索建议
  - 发布人员联想
  - 权限过滤

- 问答能力
  - `POST /api/v1/qa`
  - `POST /api/v1/qa/stream`
  - `GET /api/v1/qa` 兼容保留
  - 固定模型运行时：智谱 `glm-4.7`
  - 严格失败返回
  - `citation_status`
  - SSE 流式输出
  - 后台 `Sys Prompt` 配置
  - 后台 QA 审计
  - 结构感知切分 + 句子级拼块 + 小重叠窗口

- 部署联调
  - `frontend / backend / mysql / redis / elasticsearch / qdrant / minio` 编排已存在
  - `embedding-service` 已可运行，并可返回 `bge-m3` 向量

### 已禁用或已收口

- 管理员自由注册/切换模型
- 用户模型偏好
- 运行时从数据库动态选 chat / embedding 模型

当前 AI 运行策略是固定模型配置：chat 和 embedding 统一从 `.env` 读取。

## 当前阻塞

### 阻塞 1：QA 结果稳定性仍受上游模型影响

真实状态：

- 本地代码链路已打通
- 智谱上游仍可能出现限流或超时

影响：

- QA 成功率和时延不完全由本地代码决定

### 阻塞 2：浏览器全量回归未完成

尚未做完整人工回归的模块：

- 登录锁定与错误提示
- 个人中心改密闭环
- 搜索筛选组合
- QA 流式交互
- 后台用户/部门/Sys Prompt/QA 审计
